Ce que la démo prouve
Carte → la caisse appelle POST /api/v1/transactions (API avec clé) → la transaction existe chez vous.
Espèces → aucune transaction envoyée → sert à montrer le cas « non rapproché ».
Scanner → le ticket est transformé en image, envoyé à POST /api/mobile/ocr (OCR), puis à POST /api/mobile/verify-ticket :
  • Phase 2 doublon + règles + score de fraude
  • Phase 3 rapprochement avec la vraie transaction
Rescanner le même ticket → REJETÉ (doublon).
